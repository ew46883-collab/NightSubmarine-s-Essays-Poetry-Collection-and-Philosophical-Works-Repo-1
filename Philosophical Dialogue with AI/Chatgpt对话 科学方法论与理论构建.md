# 简要定义（一句话）
科学方法论是把“观察 → 假设 → 可证预测 → 实验/观测 → 验证/修正”这套循环化、可重复、可量化的步骤当作获取可靠知识的体系——所有结论都是暂时的、以证据为准、并且明确写出不确定性和假设前提。

# 核心步骤（工程化流程）
1. 观察与问题化：仔细记录现象，明确要回答的具体问题（可测、可反驳）。  
2. 背景与文献：把已有解释、相关数据和已有模型都读清楚，避免重复劳动。  
3. 提出假设（模型）：把问题形式化成可计算或可检验的陈述（因果关系、数学方程、机制图）。  
4. 推出可观测预测：从假设导出“如果 A 为真，则应出现 B”的具体、可量化的预测。  
5. 设计实验/观测：控制混淆变量、设置空白对照、保证可重复、尽量随机化与盲法。  
6. 收集数据并量化不确定性：使用适当测量工具并报告误差条、置信区间或后验分布。  
7. 分析与推断：用统计或贝叶斯方法评估证据强度，检验假设与替代假设的相对支持度。  
8. 报告与复现：公开方法、数据和代码，让他人能复现或反驳；基于结果修正假设，进入下一轮。

# 构建“好”理论的准则
- 可证伪性（falsifiability）：好理论应当能被实验证明为假，而不是能解释一切。正如 Karl Popper 所强调的。  
- 简洁与解释力：在解释力相近时优先更节约假设的模型（奥卡姆法则）。  
- 可推广性与泛化：不仅在当前数据成立，还能预测新情形。  
- 机制优先于相关：优先构建能解释“为什么会这样”的机理模型，而不仅仅是统计相关。  
- 数学与概念清晰：把概念形式化，便于推导预测和定量比较。  
- 对异常有容忍度：能容纳或说明反例，或明确说明适用范围。

（历史上对这些原则有系统讨论的，也包括 Francis Bacon 的经验主义传统与 Thomas Kuhn 关于范式转换的观察——理论是暂时的、科学进展并非单线。）

# 如何把“实证”做得更可靠
- 预注册研究设计，避免 p-hacking（在事后挑结果）。  
- 使用对照组、随机化与盲法来隔离因果。  
- 多方法论（三角验证）：用不同的数据源与方法检验同一结论（实验、观测、模型模拟）。  
- 量化不确定性：报告置信区间、效应量、贝叶斯后验概率，而不是只报 p 值。可用信息准则（如 AIC/BIC）或交叉验证比较模型。  
- 可重复性：公开数据与代码，或至少描述到别人能复制实验的程度。  
- 元分析与累积证据：把多项独立研究的证据合并评估总体效应。  
- 对“证据强度”做贝叶斯更新：把先验信念与新数据合并，得到更稳健的后验结论（以 Thomas Bayes 的思想为基础）。

# 如何搞清楚现象背后的“原理”／因果
- 首选随机干预（RCT）来确认因果；不能做时，用工具变量、断点回归、因果图（DAG）或因果推断框架来逼近。  
- 构建可操作的机制模型（微观到宏观的连接），用模型生成数据再与真实数据比对（模拟与对照）。  
- 做“消融实验”（逐步去掉/改变某个环节看效果），以识别必要与充分条件。  
- 关注可重复的中介变量（mechanisms），而不仅仅是终点结果。

# 常见误区与限度（要谦逊）
- 观测到相关 ≠ 因果。  
- 数据能被多种理论解释（欠定性），因此需要设计能区分理论的“危险”实验。  
- 理论永远是暂时的：它们在新证据下会被修正或替换（范式转换）。  
- 有些问题目前根本无法被现有方法直接回答（尺度、伦理、可操控性限制），需要方法学创新或跨学科合作。
